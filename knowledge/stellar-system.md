# Stellar.org Design System (dissected from live site, July 2026)

STELLAR'S COLOR SYSTEM: Near-black ink #0F0F0F on layered neutral surfaces (#FFFFFF, #F9F9F9, #F2F2F2) with one warm bone panel color #D6D3C4, and a tokenized accent set exposed as CSS custom properties: --hue-1 and --hue-4 are signature gold #FDDA24, --hue-2/5/6 are teal #00A7B5, --hue-3 is lilac #B7ACE8, plus a hot orange #FF3F00 and muted grey #969696 for secondary text and hairline borders. The lesson: build the palette as one ink, three or four quiet neutrals, and a tiny named accent map (--hue-N, --mono-N), then let a single loud accent (the gold) carry brand recognition while everything else stays calm.

STELLAR'S TYPE PAIRING: Lora (serif) is reserved exclusively for display headlines and giant stat numerals, while Inter (sans) handles all body, UI, and navigation text, mostly at font-weight 400 with occasional 500/600. Display Lora gets letter-spacing -0.03em, line-height 1.1, and text-wrap: balance; body Inter sits at line-height 1.4 to 1.6. The lesson: a serif/sans split where the serif appears ONLY at large sizes creates instant editorial contrast, and negative tracking plus balanced wrapping keeps big serif headlines tight instead of gappy.

STELLAR'S FLUID TYPE SCALE: A six-step modular scale built entirely from clamp(): --step-0 is 1rem fixed, --step-1 clamp(1.25rem, 1.2156rem + 0.1467vw, 1.333rem), --step-2 clamp(1.5625rem, ..., 1.7769rem), --step-3 clamp(1.9531rem, ..., 2.3686rem), --step-4 clamp(2.4414rem, 2.1448rem + 1.2657vw, 3.1573rem) for H2 section titles, and --step-5 clamp(3.0518rem, 2.5724rem + 2.0455vw, 4.2087rem) for the hero headline, with --step--1 and --step--2 below 1rem for captions. The lesson: define the whole type ramp as clamp() tokens (roughly a 1.25 ratio at mobile widening to 1.333 at desktop) so every heading scales fluidly with zero media queries.

STELLAR'S GIANT STAT NUMERALS: Performance stats ("$0.0007667" average transaction cost, "9.5 seconds", "90+" countries, "$200M") are set in Lora at 8rem with line-height 1, sized responsively with container query units like font-size: min(8rem, 44cqw) or min(5.5rem, 30cqw), some tinted in the bone color #D6D3C4 against dark panels. The lesson: oversized serif numbers sized with min(fixed-cap, NNcqw) is a robust pattern for stat sections because the number always fills its card without overflowing, and it turns raw metrics into the visual hero of the section.

STELLAR'S SPACING TOKENS: Two parallel spacing systems: static tokens (--spacing-static-xs 0.25rem, sm 0.5rem, md 1rem, lg 1.5rem, xl 2rem, xxxl 4rem) for component internals, and responsive tokens built from clamp() (--spacing-responsive-sm clamp(0.5rem, ..., 1rem), md clamp(1rem, ..., 1.5rem), lg clamp(1.5rem, ..., 2rem), xl clamp(2rem, ..., 3rem), xxl clamp(2rem, 1.1713rem + 3.5359vw, 4rem)) for layout gaps, with a --section-block-padding variable set to 3rem or 4rem per section. The lesson: keep fixed spacing for inside components and fluid clamp() spacing between sections, wired through overridable custom properties like --card--padding.

STELLAR'S SHAPE AND MOTION LANGUAGE: Radius is deliberately split: cards, sections, and the primary gold button (.ALML, background #FDDA24) are dead sharp at border-radius 0, small UI details get 0.25rem or 0.3em, and only form inputs and secondary CTAs go full pill at border-radius 100vh or 6.25rem with padding like 1.125rem 2rem and a 1px #969696 border. Motion uses zero cubic-bezier curves: just color 0.2s ease-in-out, transform 0.1s ease-out on hovers, background-color 250ms linear, transform 0.8s ease-in-out for big reveals, and a page-top gold progress bar animating width over 500ms ease. The lesson: sharp-cornered brand surfaces plus pill-shaped inputs reads modern-institutional, and fast default easings (100 to 250ms) with one slow 800ms reveal is enough; exotic curves are optional.

STELLAR'S HERO AND SIGNATURE MOVE: The hero is a full-bleed aria-hidden canvas element (generative WebGL-style animation, position absolute inset 0) sitting behind a rotating headline that cycles "Where blockchain meets" + "global payments" / "tokenization" / "DeFi" / "the real world" in step-5 Lora, while the real H1 is visually hidden (clip-path inset 50%) for SEO and screen readers, and the section enforces min-block-size 630px with content bottom-aligned via a grid of 1fr auto rows. The lesson: separate the accessible H1 from the animated visual headline, bottom-anchor hero copy over a live canvas, and make one kinetic element (rotating word or generative field) the site's recognizable signature.

STELLAR'S IMAGERY AND ART DIRECTION: All content images come from Sanity CDN as flat 2D illustrations and diagram-style PNGs, mostly 2048x1536 (4:3) and wide 5000px banner strips, in the same brand palette (gold, teal, lilac, bone on off-white), with tiny base64 blur placeholders inlined, functional alt text like "cross-border payments", partner logos normalized with filter: grayscale(1), and zero video, Lottie, or 3D files on the page. Notably there are no CSS gradients anywhere; every surface is flat solid color. The lesson: flat, palette-locked illustration plus grayscale logos keeps a dense fintech page cohesive, and banning gradients entirely is itself a strong stylistic rule.

STELLAR'S INFORMATION ARCHITECTURE: The homepage reads H1 "Where blockchain meets the real world", then three audience-routing cards as H3s ("For devs & builders", "For financial institutions", "For fintechs & wallets"), then H2 "Onchain updates" (news cards), H2 "Powered by performance" (the giant stats), H2 "Real use cases, live today" (payments, asset tokenization, DeFi), H2 "Grants & funding" (developer grants, enterprise fund, ecosystem), and H2 "The email you actually want to read" (newsletter) before an all-caps H3 footer sitemap. The lesson: route your three audiences immediately after the hero, prove credibility with stats and live use cases mid-page, and end with money/opportunity plus a personality-forward newsletter CTA.

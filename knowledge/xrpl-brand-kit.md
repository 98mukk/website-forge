# XRPL official brand kit (XRP Visual Guidelines v1.0) — anatomy of a professional brand system

XRPL'S BRAND ANATOMY (the 6-part skeleton every real brand kit has): 01 Messaging (value prop + signature line), 02-03 Logo systems (components, variations, clearspace, special cases, partner lockups), 04 Color palette (primary, secondary, tints/shades, ratio), 05 Typography (families, weights, type scale, pairings), 06 Visual design (pattern, diagrams). When building a brand kit for a client, follow this exact table of contents.

XRPL'S MESSAGING DISCIPLINE: one signature line ("The best way to move money around the world") plus a one-sentence value proposition ("Providing a universal asset for the exchange of value worldwide") with the key phrases underlined in the accent green. The value prop is displayed huge, white on black, with only the load-bearing words emphasized. Lesson: write the one line first; every design decision serves it.

XRPL'S COLOR SYSTEM: primary palette is only 4 values: black #000000, white #FFFFFF, one hero green #19FF83, one neutral gray #D4D7DD. A secondary neon family (blue #1AA4FF, blue-purple #791AFF, red-purple #D91AFF, magenta #FF1A8B, orange #FF671A, yellow #FAFF1A) exists but is rationed to slivers. Every hue ships with a 100-900 tint/shade scale for UI states.

XRPL'S COLOR RATIO LAW: the brand is overwhelmingly black + white + gray; the green accent gets a meaningful minority share; the six secondary neons together get only a thin strip. The ratio is documented as an actual diagram in the kit: proportions are a spec, not a vibe. Apply: ~60 percent neutral ground, ~30 percent secondary neutral, ~9 percent hero accent, ~1 percent everything else.

XRPL'S TYPOGRAPHY: Work Sans (Google Font) for everything, three weights only (Regular 400 for body, Medium 500, Bold 700 for headlines); mid weights are specified for on-screen text 14-48px, extreme weights reserved for display. Space Mono as the second face exclusively for code snippets. A two-font system where the second font has ONE job.

XRPL'S LOGO LAWS: the logo = logomark + logotype in a fixed combination, never recomposed. Black logo on light backgrounds, white on dark, no other treatments. Clearspace: the height of the X symbol on all four sides, nothing may invade it. Minimum sizes are specified per medium (96px digital at 96dpi, 192px at 192dpi, 1 inch print; symbol alone: 60px/120px/0.625in). Below minimum, switch to the symbol-only variant (favicons, app icons).

XRPL'S LOGO SYMBOLISM: the X is built from an up arrow and a down arrow (send and receive, a value exchange) with organic curves suggesting the human form (the community). Lesson for logo design: the mark should encode the product's core action, and the story should be one sentence.

XRPL'S PARTNER LOCKUP RULES (co-branding): partner logo sits beside the brand logo separated by a thin vertical divider line; spacing between them equals the X clearspace unit; horizontal lockup preferred, vertical as fallback; same black-on-light/white-on-dark rule applies to the whole lockup. A brand kit must plan for OTHER logos next to yours.

XRPL'S PATTERN SYSTEM: the logomark's curves are exploded into a repeatable decorative pattern (waves and chevrons built from the X anatomy) used as brand texture on dark grounds. Lesson: derive decoration from the logo geometry itself so even abstract patterns stay on-brand.

BRAND KIT AS A PRODUCT LESSON (for Lake Shore Digital): this whole document is what companies pay agencies to produce: messaging, logo laws, palette with ratios, type system, patterns, partner rules. The Forge pipeline (research agent, branding agent, build agent) should output exactly this artifact class for clients, then build the site FROM it.
